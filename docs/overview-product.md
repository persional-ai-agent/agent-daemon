# Agent Daemon 产品概览

> 文档元数据
> - 文档版本：v1.0.0
> - 最后更新：2026-05-16
> - 更新来源：[docs/dev/1-task-docs-baseline.md](dev/1-task-docs-baseline.md)

## 1. 产品定位

- 目标用户：需要在本地工作区运行 AI Agent 的开发者与自动化运维使用者。
- 核心问题：在单机环境中统一管理会话、工具调用、网关接入、任务调度与可视化操作。
- 核心价值：同一引擎支持 CLI、HTTP/WebSocket API、Web UI、TUI 与多平台消息网关接入。
- 非目标：不提供托管云服务与多租户 SaaS 管理面。

## 2. 功能边界

- 核心功能：
  - Agent 对话执行（同步、流式、WS）。
  - 会话持久化、摘要压缩、分页查看与重放。
  - 内置工具、工具集管理、审批流、插件扩展。
  - 网关接入（Matrix/Feishu/DingTalk/WeCom/Mattermost/SMS/Telegram/Discord/Slack/WhatsApp/Signal/Email/Webhook/HomeAssistant/BlueBubbles/Yuanbao）。
  - Cron 定时任务（agent/script 两种执行模式，支持投递、重试与链式上下文）。
  - Web Dashboard 与独立 TUI 操作端。
- 不支持功能：
  - 分布式多节点调度与跨实例共享状态。
  - 内置完整 RL/Spotify/Yuanbao 工具实现（当前为占位或门禁响应）。
- 关键对象：Session、Message、Tool、Approval、CronJob、CronRun、Gateway Platform、Plugin、Skill。
- 关键状态：会话运行/取消、审批待确认/已确认、网关启用/停用、定时任务暂停/运行、模型主提供方/回退提供方。

## 3. 关键场景

| 场景 | 用户目标 | 成功标准 | 异常/边界 |
|------|----------|----------|-----------|
| 本地对话执行 | 在指定会话中发起 Agent 任务 | 返回最终响应并持久化消息 | 支持取消、超时、恢复 |
| 多入口运维 | 通过 CLI/Web/TUI 管理会话、工具、配置与网关 | 各入口访问同一后端能力 | 前端入口可部分降级但后端接口保持可用 |
| 外部消息接入 | 从 IM/Webhook 接收消息并转发到 Agent | 平台事件入站后触发会话执行 | 平台未配置或鉴权失败时返回明确错误 |
| 定时自动化 | 定时执行 prompt 或脚本并记录结果 | 任务按计划触发，run 可追踪 | 失败可重试；支持按条件投递结果 |
| 插件扩展 | 安装并启用外部工具/提供方/命令 | Manifest 校验通过并可被发现 | 校验失败或禁用时不可执行 |

## 4. 核心流程

```text
1. 用户通过 CLI/Web/TUI/网关发送输入，指定或创建 session_id。
2. Agent 引擎加载会话历史与系统提示，执行模型调用与工具编排。
3. 结果写入会话存储，并通过当前入口返回；若走网关可触发回发与事件钩子。
4. 会话可在后续被压缩、重放、导出或用于 Cron 链式上下文。
```

## 5. 产品规则

- 权限规则：高风险工具调用走审批机制；审批有 TTL 并绑定会话/命令范围。
- 状态流转：会话支持继续、取消、分支、重命名、删除与导出；Cron 任务支持暂停/恢复/触发/删除。
- 异常处理：API 返回结构化错误码；网关未接入或未实现时返回可识别错误。
- 兼容约束：多入口复用同一 API 语义；平台命令规范化保证网关指令一致。
- 用户可见行为：Web 与 TUI 可查看会话、工具、技能、模型、网关、Cron、审批与诊断信息。

## 6. 非功能要求（按需）

- 性能：支持流式响应与会话压缩；会话查询支持分页与检索。
- 可用性：提供 health 接口、取消接口、恢复机制与诊断端点。
- 安全：平台入站支持密钥校验；插件支持签名/校验和验证；审批控制高风险操作。
- 兼容性：支持多模型提供方与回退策略；支持多种网关平台协议。
- 可观测性：保留会话、Cron run、网关事件与诊断导出能力。
