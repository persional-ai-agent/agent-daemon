# 009 调研：会话级审批状态补齐

## 背景

当前项目已有：

- hardline 永久阻断
- `requires_approval` 单次显式放行

但仍缺少会话级审批状态，导致同一会话中的多个危险命令需要反复携带审批参数，交互成本较高。

## 本次范围

- 新增会话级审批状态存储（内存态 + TTL）
- 新增 `approval` 工具：
  - `status`
  - `grant`
  - `revoke`
- terminal 在危险命令判定时读取会话审批状态

不纳入：

- 跨进程持久化审批状态
- 用户侧 UI 交互确认流
- 命令级细粒度审批策略

## 结论

该方案可在不改变现有工具协议和 API 结构的前提下，实现最低成本的交互式审批状态闭环。
